    HTML5:

    Site pessoal para treinar HTML5 e CSS e expor meus conhecimentos, com minhas considerações sobre o código, erros e melhorias!

    Página Principal

    Comentários

    1. Considerações iniciais: Optei por um layout simples estilo "Beacons" para que fosse mais fácil tornar o site responsivo.

    2. Erros: Primeiramente adicionei a imagem de fundo pela tag "STYLE", esse erro foi corrigido, agora a imagem de fundo foi incluida pelo "CSS" para facilitar a configuração.

    3. Melhorias: Usei logo de cada linguagem para criar um link para suas respectivas páginas, porem não consegui separar o texto da tag "a" e alinhar o logo no começo e o texto no centro do botão. Preciso aprender a tag correta para futuramente corrigir o problema no "CSS".

    Página Python

    Comentários

    1. Considerações: Novamente optei por um layout simples, para que fosse fácil tornar o site responsivo.

    2. Erros: Primeiramente tentei utilizar as tags "PRE" e "CODE" para disponibilizar o código, porem tive dificuldade para organizar as informações no "CSS".

    3. Melhorias: Utilizei a tag "BR" para fazer a divisão do texto "intro" que deveria ter sido feita pelo "CSS", esse problema já foi corrigido, pois essa não é a utilização correta da tag "BR".

    Comentários

    1. Considerações: Utilizei uma tag "DIV" para criar um container e outra para criar uma divisão entre a imagem e o texto, não tenho certeza se essa é a tag certa para realizar essa função.

    Comentários

    1. Considerações: Mantive a lista com um item apenas, para facilitar a configuração no "CSS" e manter um padrão no "HTML" que fosse fácil e intuitivo de revisar.

    Considerações Finais:

    O objetivo do site era treinar meus conhecimentos de CSS e HTML na prática. Os problemas que encontrei ajudaram a delimitar meus pontos fracos e mais importante me ajudaram a desenvolver minhas habilidades de resolver problemas.

    Também percebi ao longo do projeto que a resolução primaria para os problemas é criar "gambiarras o que acaba por criar problemas futuros. A solução é um planejamento maior no início do projeto.
